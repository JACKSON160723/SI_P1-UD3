Práctica 1. Exploración del sistema de archivos y estructura de directorios en Windows 11

En esta práctica realizarás una exploración inicial de los sistemas de archivos, las herramientas de gestión (gráficas y por comandos) y la estructura de directorios en un entorno Windows 11 Pro. Conocerás cómo identificar el sistema de archivos del disco, navegar por las carpetas principales y utilizar herramientas como el Explorador de archivos y PowerShell para obtener información detallada.

Objetivos de la práctica

  • Identificar el sistema de archivos del equipo.
  • Gestionar archivos y directorios mediante herramientas gráficas y PowerShell.
  • Reconocer la estructura de directorios del sistema.
  • Comprender la función de las carpetas principales.

Ejercicio 1. Identificación y análisis del sistema de archivos

Parte A. Comprobación con interfaz gráfica

  1. Prueba 5 formas diferentes de acceso al Explorador de archivos.
Explorador de archivos

El Explorador de archivos es la herramienta gráfica de Windows que permite navegar, organizar y gestionar archivos, carpetas y unidades del sistema. Ofrece una vista jerárquica del sistema de archivos y acceso rápido a ubicaciones frecuentes como Este equipo, Descargas o Documentos.

Existen diversas formas de acceder al Explorador de archivos:

  1. Icono de la barra de tareas: Haz clic en el icono del Explorador de archivos anclado en la barra de tareas.
  2. Atajo de teclado: Pulsa Windows + E.
  3. Menú Inicio (búsqueda): Abre el Menú Inicio, escribe Explorador de archivos y pulsa Enter.
  4. Menú Inicio (acceso directo): Abre el Menú Inicio. Si aparece el icono de Explorador de archivos en la parte superior o lateral, haz clic sobre él.
  5. Menú contextual avanzado (Win + X): Pulsa Windows + X y selecciona Explorador de archivos en el menú.
  6. Cuadro de Ejecutar (Win + R): Pulsa Windows + R, escribe explorer y pulsa Enter. También puedes escribir una ruta, por ejemplo: explorer C:\.
  7. Desde una terminal (PowerShell o CMD): Abre PowerShell o Símbolo del sistema, escribe explorer y pulsa Enter para abrir la carpeta actual. O bien: explorer C:\Ruta\A\Carpeta.
  8. Desde el Administrador de tareas: Abre el Administrador de tareas (Ctrl + Shift + Esc), menú Archivo → Ejecutar nueva tarea, escribe explorer y pulsa Enter.
  9. Doble clic en accesos directos del escritorio: Si tienes accesos directos como Este equipo, Carpeta personal u otras carpetas en el escritorio, haz doble clic sobre ellos.
  1. Una vez estés en el Explorador de archivos, identifica el panel izquierdo y selecciona Este equipo.
  2. Localiza la unidad C: (unidad del sistema).
  3. Haz clic derecho sobre C: → Propiedades.
  4. Busca el campo Sistema de archivos e identifica si es NTFS, FAT32, etc.
  5. Justifica por qué Windows 11 utiliza ese sistema de archivos.

Parte B. Comprobación mediante PowerShell

  1. Prueba 5 formas diferentes de acceder a PowerShell.
PowerShell

PowerShell es una consola avanzada de administración que permite ejecutar comandos, automatizar tareas mediante scripts y gestionar prácticamente cualquier aspecto del sistema operativo.

Existen diversas formas de abrir PowerShell en Windows 11:

  1. Menú Inicio (búsqueda rápida): Abre el Menú Inicio, escribe PowerShell, pulsa Enter o haz clic en Windows PowerShell.
  2. Atajo de teclado (Win + X): Pulsa Windows + X para abrir el menú avanzado y selecciona Windows PowerShell o Windows PowerShell (Administrador).
  3. Cuadro de Ejecutar (Win + R): Pulsa Windows + R, escribe powershell y pulsa Enter. Para abrirlo como administrador: 
    powershell -Command "Start-Process powershell -Verb runAs"
  4. Desde el Explorador de archivos: Navega a cualquier carpeta, haz clic en la barra de direcciones, escribe powershell para abrirlo en esa ruta.
  5. Desde una ventana CMD:
    CMD (Símbolo del sistema): consola antigua basada en texto, menos potente que PowerShell.
    Si tienes abierto CMD, escribe: powershell. Para abrirlo como administrador: 
    powershell Start-Process powershell -Verb runAs
  6. Desde el Administrador de tareas: Abre el Administrador de tareas (Ctrl + Shift + Esc), ve a Archivo → Ejecutar nueva tarea, escribe powershell y mark Crear esta tarea con privilegios administrativos.
  7. Desde Windows Terminal:
    Windows Terminal: aplicación moderna que puede abrir varias consolas (PowerShell, CMD, WSL…). No es una consola, sino un contenedor de consolas.
    Abre Windows Terminal desde el menú Inicio o Win + X y selecciona PowerShell en una nueva pestaña.
  8. Acceso directo en barra de tareas o escritorio: Puedes anclar PowerShell a la barra de tareas o crear un acceso directo.
  1. Abre la aplicación Windows PowerShell.
  2. Ejecuta el siguiente comando:
Get-Volume
¿Qué es Get-Volume?

Get-Volume es un cmdlet de PowerShell. Un cmdlet (command-let) es un comando especial de PowerShell, más avanzado que los comandos tradicionales de CMD. Trabajan con objetos, permitiendo obtener información estructurada y completa.

¿Qué hace?

Muestra información detallada sobre los volúmenes del sistema (unidades lógicas con sistema de archivos asignadas a una letra).

¿Para qué sirve? Permite identificar rápidamente:

  • El sistema de archivos de cada volumen.
  • El tamaño total de la unidad y el espacio libre disponible.
  • El estado de salud del volumen (HealthStatus).
  • La letra o punto de montaje y el tipo de unidad (Fixed, Removable, etc.).

Información típica que devuelve:

  • DriveLetter → C:, D:, E:
  • FileSystem → NTFS, FAT32…
  • Size → tamaño total del volumen
  • SizeRemaining → espacio libre
  • HealthStatus → estado ("Healthy", "Warning", etc.)
  • DriveType → Fixed, Removable, Optical…
  1. Localiza la fila correspondiente a la unidad C: y revisa: FileSystem, Size, SizeRemaining y HealthStatus.
  2. Comprueba que el sistema de archivos coincide con el visto en la interfaz gráfica.

Ejercicio 2. Gestión del sistema de archivos mediante herramientas gráficas

  1. Abre el Explorador de archivos.
  2. Activa Ver → Mostrar → Elementos ocultos para visualizar carpetas del sistema.
  3. Explora las siguientes rutas una por una:
    • C:\
    • C:\Windows
    • C:\Program Files
    • C:\Users
    • C:\ProgramData
  4. En cada carpeta: observa su contenido, identifica qué tipo de archivos/subcarpetas contiene y trata de deducir su función general.
  5. Registra brevemente la función de cada directorio principal.

Ejercicio 3. Gestión del sistema de archivos mediante comandos (PowerShell)

  1. Abre PowerShell.
  2. Ejecuta el comando: 
    Get-ChildItem C:\
  3. Observa qué elementos aparecen en la raíz del sistema.
  4. Ejecuta ahora: 
    Get-ChildItem C:\Windows -Directory
  5. Analiza las carpetas mostradas y compáralas con lo visto en el Explorador.
  6. Ejecuta: 
    Get-Item C:\Users
  7. Observa los metadatos que aparecen (tamaño, atributos, etc.).
  8. Compara las diferencias entre lo que se ve en la GUI y lo que proporciona PowerShell.

Ejercicio 4. Estructura de directorios: análisis y funciones

  1. Localiza y examina estas carpetas clave:
    • C:\Windows → archivos esenciales del sistema operativo.
    • C:\Program Files → aplicaciones instaladas de 64 bits.
    • C:\Program Files (x86) → aplicaciones de 32 bits.
    • C:\Users → perfiles y datos de usuario.
    • C:\ProgramData → datos compartidos y configuraciones globales.
  2. Describe qué función cumple cada carpeta.
  3. Indica qué tipo de información contiene cada una (Archivos del sistema, Programas instalados, Configuraciones, Datos de usuario, Datos globales o compartidos).

Ejercicio 5. Rutas absolutas y relativas

  1. Abre PowerShell.
  2. Ejecuta: 
    Get-Location
  3. Anota la ruta actual.
  4. Cambia de carpeta usando: 
    Set-Location C:\Windows
  5. Vuelve al directorio anterior con: 
    Set-Location ..
  6. Vuelve a ejecutar: 
    Get-Location
  7. Explica:
    • Qué es una ruta absoluta.
    • Qué es una ruta relativa.
    • Qué ejemplos has utilizado en este ejercicio.
